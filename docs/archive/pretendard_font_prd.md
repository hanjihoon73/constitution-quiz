# 전역 폰트(Pretendard) 세팅 및 가독성 개선 기획서 (PRD)

## 1. 목적
현재 시스템 폰트(OS별 상이: 맑은 고딕, Apple SD 산돌고딕, Roboto 등)로 인해 기기와 플랫폼 간 일관성이 없고 안드로이드·윈도우 랩탑 환경에서 가독성이 다소 떨어지는 문제가 있습니다.
이를 개선하여 모던웹 디자인 트랜드에 가장 어울리고 최적인 **Pretendard (프리텐다드)** 폰트를 서비스 전역에 적용, 보다 깔끔하고 가독성 높은 UI를 제공합니다.

## 2. 요구사항
- **웹 폰트 속도로드 최적화**: 별도의 무거운 폰트 시스템이나 로컬 파일(.woff2) 다운로드 작업을 거치지 않고 가장 빠르고 안정적인 CDN 방식을 이용 (`jsdelivr`의 `pretendardvariable.min.css`)
- **코드 일관성**: Tailwind 및 기존 Next.js의 폰트 변수(`--font-sans`)의 작동 원리를 해치지 않게 `body`의 `font-family`를 최상단 선순위로 오버라이드.
- **안정성**: 만에 하나, 웹 폰트 로드가 안될 경우를 대비해 기존 Fallback Font 설정을 그대로 뒤에 배치.

## 3. 적용 대상 범위
- 서비스 전체 페이지 및 모든 텍스트 엘리먼트 (Heading, Paragraph, 버튼 등 전부)
