# 퀴즈팩 순서 변경 시나리오 분석 (DB 및 상태 로직 기준)

현재 시스템 (`src/lib/api/quizpacks.ts` 의 `getQuizpacksWithStatus` 함수)은 사용자의 퀴즈팩 상태를 실시간 동적으로 계산합니다. 기본적으로 **`user_quizpacks` 테이블에 저장된 개별 진행 이력(in_progress, completed)**이 우선 반영되며, 이력이 없는 퀴즈팩의 접근 권한은 **"현재 로드맵의 직전 퀴즈팩 완료 여부"**에 따라 즉각 재계산됩니다.

아래는 퀴즈팩 순서(Order)를 어드민에서 변경했을 때 발생할 수 있는 5가지 케이스별 영향도입니다.

---

### 1. 가입 후 로그인은 했지만 퀴즈팩을 1개도 시작하지 않은 경우
- **분석 결과**: 시스템 운영에 가장 영향이 없습니다.
- **상태 변화**:
  - 사용자 DB 이력(`user_quizpacks`)이 전무하므로, 모든 퀴즈팩의 상태는 순서(Order)에 전적으로 의존합니다.
  - 순서를 어떻게 바꾸든, "새롭게 1번째로 지정된 퀴즈팩"만 `opened` 상태가 되며, 나머지 2번~끝번 퀴즈팩은 모두 `closed` 상태가 됩니다.
  - 사용자는 변경된 새로운 첫 퀴즈팩부터 정상적으로 시작하게 됩니다.

### 2. 순서를 변경한 퀴즈팩이 닫힘 상태(closed)인 경우
- **조건**: 사용자가 아직 해금하지 못한 상태 (이전 퀴즈팩을 완료하지 못함 & 스스로 진행한 이력 없음).
- **상태 변화**:
  - 이동하는 위치에 따라 상태가 바뀔 수 있습니다.
  - **1번 위치로 이동할 경우**: 즉시 `opened` 상태로 잠금이 해제됩니다.
  - **이미 완료한(completed) 퀴즈팩 바로 다음 위치로 이동할 경우**: 직전 퀴즈팩 완료 조건이 충족되어 즉시 `opened` 상태로 잠금이 해제됩니다.
  - 그 외 미완료 퀴즈팩 뒤로 이동할 경우: 여전히 `closed` 상태를 유지합니다.

### 3. 순서를 변경한 퀴즈팩이 열림 상태(opened)인 경우
- **조건**: 아직 문제를 풀기 시작하지는 않았으나 첫 번째 퀴즈팩이거나 바로 앞 퀴즈팩을 완료해서 `opened`가 된 상태. (`user_quizpacks` 에 기록이 없는 상태)
- **상태 변화**:
  - **뒤로 이동하는 경우**: 이동한 새 위치의 ‘직전 퀴즈팩’이 미완료 상태라면, **`opened`였던 퀴즈팩이 돌연 `closed` 상태로 닫히게 됩니다.** 사용자는 열려있던 퀴즈를 갑자기 시작할 수 없게 됩니다.
  - **참고**: 사용자가 이미 한 문제라도 풀어서 DB에 `in_progress` 등으로 기록이 생성된 상태라면, 아래 4번 사례가 적용되어 닫히지 않습니다.

### 4. 순서를 변경한 퀴즈팩이 진행 중 상태(in_progress)인 경우
- **조건**: 사용자가 이미 문제를 풀고 있는 중이라 DB에 진행 이력이 남은 상태.
- **상태 변화**:
  - 해당 퀴즈팩 자체는 DB 기록이 우선하므로, 어디로 순서를 이동하든 **계속 `in_progress` 상태를 확고히 유지**합니다. 사용자는 이동한 위치에서도 퀴즈를 문제없이 이어서 풀 수 있습니다.
  - **UI 상의 어색함 발생 가능성**: 미완료(`closed`)된 퀴즈팩들 사이에 이 퀴즈팩이 끼어들 경우, 사용자 화면에서는 "1, 2번은 잠겨(closed)있는데 3번이 진행 중(in_progress)인" 논리적 순서 역전 현상이 노출됩니다.

### 5. 순서를 변경한 퀴즈팩이 완료 상태(completed)인 경우
- **조건**: 사용자가 모든 하위 퀴즈를 다 풀어서 DB에 `completed` 처리된 상태.
- **상태 변화**:
  - 이 퀴즈팩 자체는 어디로 이동하든 **계속 `completed` 로 유지**됩니다.
  - **파급 효과 (다음 퀴즈팩 자동 오픈)**: 이 퀴즈팩이 이동하여 정착한 위치의 **"바로 다음 순서 퀴즈팩"을 즉시 `opened` 상태로 연쇄 잠금 해제**시킵니다.
  - 반대로, 기존에 이 완료된 퀴즈팩 바로 뒤에 있어서 잠금이 해제(`opened`)되어 있던 원래 퀴즈팩은, 앞에 미완료 퀴즈팩이 배치되면서 갑자기 `closed`로 닫힐 위험성이 큽니다 (이미 시작하지 않은 경우).

### 💡 관리자 운영 정책 제언
위 분석에 따라, **앱이 이미 운영 중이고 사용자가 활동하고 있는 도중에는 기존 퀴즈팩의 순서를 섣불리 섞거나 끼워 넣는 행위는 권장되지 않습니다.**
*   (O) 가장 안전한 방식: 완전히 새로운 퀴즈팩을 **목록의 가장 마지막**에 추가(`closed` 상태로 배포)
*   (X) 지양해야 할 방식: 기존 진행 중이거나 완료된 퀴즈팩의 순서를 뒤바꾸어, 기존 해금(`opened`) 체계를 혼란에 빠뜨리는 구조 변경
