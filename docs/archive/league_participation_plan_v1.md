# 리그 참여 기준 변경 개발 계획서 (v1)

## 1. 개요
* **작업 목표**: 기획 변경점(XP 1 이상 획득 시에만 랭킹 반영 및 리그 참가 판별)을 충족하기 위해 데이터베이스 함수와 프론트엔드의 로직을 동기화.
* **대상 파일**: `src/lib/api/league.ts` 및 Supabase Database RPC (SQL)

## 2. 프론트엔드 구현 (`src/lib/api/league.ts`)
#### 유지보수 (`checkLeagueParticipation`)
- 기존 로직인 `user_quizpacks` 테이블에서 이번 주 활동 내역을 추적하는 방식을 그대로 유지합니다. (피드백 수용: 퀴즈 재풀이로 XP를 0점 획득했어도 새로운 주간 참가 팝업이 뜨는 사용자 경험 고도화 의도).

## 3. 백엔드 구현 계획 (Supabase SQL)
#### [MODIFY] `get_weekly_ranking` 기능 (RPC)
- Supabase MCP를 통해 함수 코드를 확인하여, 베이스 쿼리의 랭킹 산정 과정(`FROM users`)에 `WHERE weekly_xp > 0` 조건을 삽입하는 마이그레이션을 적용합니다.
- 데이터 노이즈를 방지하고 "이번 주 활성 참가자가 0명입니다" UI가 원활히 작동하도록 만듭니다.

2. 퀴즈 실행: 퀴즈팩을 완료하여 1 이상의 XP 획득 후 메인 진입 시 "리그 시작(참여 완료)" 팝업 노출 검증.
