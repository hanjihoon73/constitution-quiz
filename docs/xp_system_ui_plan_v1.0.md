# XP 시스템 UI 구현 계획 (v1.0)

본 문서는 구현된 XP 시스템(백엔드/DB)을 바탕으로 프론트엔드 UI를 구현하기 위한 개발 계획서입니다. 첨부된 디자인 시안과 요구사항을 충족하도록 애니메이션과 컴포넌트 배치를 설계했습니다.

## 1. 개요
사용자가 퀴즈를 풀면서 획득/차감되는 XP를 직관적으로 확인할 수 있도록 앱 곳곳에 UI를 추가합니다. 숫자가 자연스럽게 올라가거나 내려가는 모션(카운트업/다운)을 적용하여 동적인 사용자 경험을 제공합니다.

## 2. 공통 애니메이션 훅 및 컴포넌트 개발 (`src/hooks/useCountUp.ts`)
- 외부 라이브러리 없이 가벼운 커스텀 훅(`useCountUp`)을 개발하여 애니메이션 제어
- `start`, `end`, `duration` 파라미터를 받아 `requestAnimationFrame` 기반의 부드러운 숫자 변경 효과 제공
- `XpStatusModal`, `QuizLayout` 헤더 등 여러 곳에서 재사용

## 3. 화면별 구현 계획

### 3.1. 홈 화면 & 마이페이지 (XP 보유 현황 모달)
- **위치**: 헤더(`Header.tsx`) 바로 아래 중앙 고정 (스크롤과 무관하게 표시되도록 `fixed` 또는 뷰포트 기반 플로팅 처리)
- **데이터**: `users.total_xp` (전체 보유 XP)
- **디자인**:
  - 주황색(#ff8400) 배경, 두꺼운 검은색 테두리(#2d2d2d)의 캡슐 모양 모달
  - 흰색 텍스트 (`000000 XP`)로 우측 정렬
  - 은은한 드롭섀도우(Drop Shadow) 적용
- **동적 연출 (카운트업 애니메이션)**:
  - 퀴즈팩 마지막 문제를 체크한 후(완료 화면을 거치지 않고) 홈으로 바로 나왔을 때 보유 XP가 올라가는 효과
  - 이를 위해 `useAuth` 또는 최상위 상태(예: `sessionStorage`)에 '이전 total_xp'를 기억해두었다가 홈 마운트 시 비교하여 카운트업 실행

### 3.2. 퀴즈 진행 화면 (`QuizLayout.tsx` 및 `useQuiz.ts`)
- **위치**: 퀴즈 화면 최상단 헤더 내부, 좌측 '나가기(ArrowLeft)' 버튼과 같은 행의 맨 우측
- **데이터**: 현재 진행 중인 퀴즈팩의 `pending_xp`
- **상태 관리**: `useQuiz` 훅 내부에 `pendingXp` 상태 추가 및 API 성공 후 업데이트
- **애니메이션**:
  - 정오답 체크 모션(정답화면 표시) 후 약간의 시간차를 두기 위해 **300ms 지연** 적용
  - 획득 시 카운트업 (예: `+100 XP`)
  - 힌트 사용 등 차감 시 카운트다운 적용

### 3.3. 퀴즈팩 완료 화면 (`[packId]/complete/page.tsx`)
- **위치**: 결과 요약 카드 중앙 (기존: `총 퀴즈 / 정답 / 오답` -> 변경: `퀴즈수 / 정답수 / 획득 XP`)
- **데이터**: 해당 퀴즈팩에서 최종 확정된 `user_quizpacks.earned_xp`
- **애니메이션**: 정답률(원형 차트 텍스트) 카운트업이 실행되는 **완전히 동일한 타이밍**에 획득 XP 숫자 애니메이션 시작

### 3.4. 홈 화면 퀴즈팩 카드 (`QuizpackCard.tsx`)
- **위치**: 홈 화면의 진행중(in_progress) 또는 완료(completed) 상태인 퀴즈팩 카드 우측 하단
- **데이터**:
  - 진행중: `user_quizpacks.pending_xp`
  - 완료: `user_quizpacks.earned_xp`
- **디자인/레이아웃**:
  - 기존 우측 하단의 별점 영역을 왼쪽의 '퀴즈 n개' 텍스트 바로 우측으로 이동 (퀴즈 n개와 별점 사이에 얇은 회색 선을 두어 구분)
  - 기존 별점이 있던 우측 하단 자리에 XP 수치 표시 (예: `150 XP`)
  - 애니메이션 효과 없이 고정 텍스트로 표시
- **API 수정**: `src/lib/api/quizpacks.ts` 쿼리 수정하여 `pending_xp`와 `earned_xp` 조회 추가

## 4. 진행 순서 (EXECUTION)
1. `useCountUp.ts` 훅 및 `XpBadge.tsx` (모달/라벨 공통 텍스트 컴포넌트) 작성
2. `quizpacks.ts` (홈 불러오기 API) 반환 타입에 xp 필드 추가 (`QuizpackWithStatus`)
3. `QuizpackCard.tsx` 레이아웃 수정 (별점 위치 변경 및 XP 표시)
4. 글로벌 XP 상태 모달 개발 및 `page.tsx`, `profile/page.tsx` 연동
5. `useQuiz.ts` 훅과 `QuizLayout.tsx`를 수정하여 퀴즈 진행 화면 상단에 300ms 지연 애니메이션이 적용된 XP 증감 표시 추가
6. 퀴즈팩 완료 화면 요약 카드 레이아웃 수정 및 카운트업 타이밍 동기화

## 5. 검증 계획 (VERIFICATION)
- **수동 검증 항목**:
  1. 앱 접속 시 홈 화면과 마이페이지 상단 중앙에 드롭섀도우가 있는 XP 모달이 정확한 위치에 뜨는지 확인.
  2. 홈 퀴즈팩 카드의 진행중/완료 상태에 따라 하단 별점이 이동하고 우측에 XP 수치가 노출되는지 대조.
  3. 퀴즈 진입 후 정답 선택 시 **300ms 지연 후** 우측 상단의 `+XX XP` 카운트업이 나타나는지 체크.
  4. 힌트 사용 시 포인트가 차감(카운트다운) 되는 부드러운 전환 확인.
  5. 퀴즈 완료 화면 진입 시 중앙 요약 카드에 '획득 XP'가 표시되며 정답률 차트와 타이밍이 일치하는지 확인.
  6. **(중요)** 마지막 퀴즈 정오답 후 홈 화면으로 바로 빠져나왔을 때 상단 모달의 전체 XP가 증가하는 이펙트 정상 작동 확인.
