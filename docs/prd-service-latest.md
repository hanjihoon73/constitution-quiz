# 모두의 헌법 — 서비스 통합 PRD

> **문서 버전**: v1.0 통합본
> **작성 기준**: 개발 과정에서 작성된 모든 기획 문서를 취합하여 현재 구현 상태를 기준으로 정리
> **Tech Stack**: TypeScript / Next.js App Router / Supabase / Tailwind CSS / shadcn/ui / Lucide React

## 1. 서비스 개요

### 1.1 개발 배경

법은 모르면 칼이 되어 나를 겨누고, 알면 유용한 도구가 되는 양면성을 지닌다. 헌법은 대한민국의 최상위 법률이자 국가의 기본 틀을 규정하는 핵심 법으로 국민이라면 누구나 관심을 가져야 하지만, 방대한 양과 어려운 용어로 인해 관련 종사자나 일부를 제외하면 관심을 갖지 않는다. 누구나 쉽고 재미있게 헌법을 알아가고 가까이 할 수 있도록 헌법을 활용한 캐주얼 퀴즈 서비스를 개발한다.

### 1.2 서비스 기본 정보

- **서비스명**: 모두의 헌법
- **콘텐츠**: 1,000개 이상의 대한민국 헌법 관련 퀴즈를 쉬운 단계부터 어려운 단계로 풀어나가며 헌법 지식을 쌓는 캐주얼 퀴즈 서비스

### 1.3 디바이스 & 레이아웃

- **타겟 디바이스**: 모바일 전용 반응형 웹앱 (Mobile-only Responsive Web App)
- **레이아웃 방식**: PC에서도 모바일 비율을 유지하는 'Centered Mobile Frame' 구조
  - Max-width: `480px`
  - PC: 가로 중앙 정렬, 외부 배경 `#ffffff`, 컨테이너에 `shadow-2xl`
  - Height: `min-height: 100dvh` (주소창 대응)
  - 내부 레이아웃: `flex flex-col`
- **iOS Safari 대응**: 고무줄 스크롤 방지(`overscroll-behavior: none`), Safe Area 대응(`env(safe-area-inset-top)`), 터치 하이라이트 제거(`-webkit-tap-highlight-color: transparent`)
- **Overlay 제한**: Modal, Drawer 등 모든 Overlay 컴포넌트는 `480px` 프레임 내부에 렌더링

### 1.4 비주얼 디자인

- **컨셉**: 모던하고 세련된 디자인, 글래스모피즘
- **테마**: Light
- **Primary**: `#2D2D2D` (다크 그레이)
- **Secondary / Accent**: `#FF8400` (오렌지)
- **Background**: `#ffffff`
- **정답 색상**: `#38D2E3` (Cyan)
- **오답 색상**: `#FB84C5` (Pink)

## 2. 인증 & 사용자 관리

### 2.1 로그인

- 구글 OAuth 또는 카카오 OAuth 로 로그인
- 최초 로그인 시 `users` 테이블에 신규 사용자 생성 (닉네임 제외)
- 로그인/로그아웃 이력은 `user_login_history` 테이블에 저장 (action: 'login' / 'logout')

### 2.2 온보딩 (닉네임 설정)

- 최초 로그인 후 닉네임이 없으면 `/onboarding`으로 이동
- 닉네임 정규식: `^[가-힣a-zA-Z0-9]{4,20}$` (한글/영문/숫자, 4~20자)
- 다른 사용자와 중복 불가, 실시간 중복 조회
- 온보딩 페이지에서 브라우저 뒤로가기 시 로그아웃 안내 팝업 표시 → 확인 시 로그아웃 후 `/login`으로 이동 (온보딩 이탈 방지)
- 온보딩 완료 후 홈 화면 진입 시 `WelcomeDialog` + 콘페티 효과 표시

### 2.3 마이페이지 (프로필)

- 닉네임 수정 (실시간 중복 확인)
- 누적 XP 및 타이틀 표시
- 로그아웃
- 회원탈퇴: `users` 테이블 사용자 정보 즉시 삭제 + 소셜 계정 연동 해제. `user_quizzes`, `user_quizpacks`, `user_quizpack_ratings`, `user_login_history` 데이터는 삭제하지 않음

### 2.4 페이지 이동 규칙

| 시나리오 | 동작 |
|----------|------|
| 로그인 → OAuth 콜백 | 닉네임 없으면 `/onboarding`, 있으면 `/` |
| 온보딩 → 브라우저 뒤로가기 | 로그아웃 안내 팝업 → 확인 시 로그아웃 후 `/login` |
| 홈 → 브라우저 뒤로가기 | 자연 흐름 허용 (세션 유지) |
| 홈 → 헤더 프로필 아이콘 | `/profile`로 이동 |
| 마이페이지 → 로그아웃 | 로그아웃 후 `/login` |
| 마이페이지 → 나가기 | 홈으로 이동 |
| 홈/마이페이지 → 헤더 BI 클릭 | `/`로 이동 |
| 마이페이지 → 회원탈퇴 | 안내 팝업 → 탈퇴 확인 → 처리 후 `/login` |

## 3. 홈 화면

### 3.1 퀴즈팩 목록

- 사용자의 퀴즈팩 로드맵(`user_quizpacks`)에 따라 퀴즈팩 카드 목록 표시
- 홈 진입 시마다 `current_quizpack`(진행 중 또는 열림 상태의 퀴즈팩)으로 자동 포커싱 및 강조

### 3.2 퀴즈팩 상태

| 상태 | 설명 |
|------|------|
| `closed` | 잠금 상태 — 직전 퀴즈팩 미완료. 시작 불가 |
| `opened` | 잠금 해제 — 시작 대기 중 |
| `in_progress` | 진행 중 |
| `completed` | 완료 |

- 1번 퀴즈팩만 최초에 `opened`, 나머지는 `closed`
- 직전 퀴즈팩 완료 시 다음 퀴즈팩 자동 `opened`
- `closed` 상태에서 시작 시도 시 잠금 안내
- `in_progress` 상태에서 다른 퀴즈팩 시작 시 기존 진행 기록 초기화 안내 팝업 표시
- 진행 중 퀴즈팩과 열림 퀴즈팩은 각각 1개씩만 존재 가능

### 3.3 퀴즈팩 카드 표시 정보

- 퀴즈팩 번호, 키워드 태그, 상태 표시
- 진행 중(`in_progress`): `pending_xp` 표시
- 완료(`completed`): `earned_xp` 표시
- 전체 사용자 별점 평균 (`quizpack_statistics.average_rating`)

### 3.4 모든 퀴즈팩 완료 시

- 마지막 퀴즈팩 완료 후 홈으로 이동 시 전체 완료 축하 팝업(`AllClearDialog`) + 콘페티 효과

### 3.5 리그 플로팅 버튼

- 위치: 홈, 마이페이지
- 스타일: 원형 뱃지, 배경 `#2D2D2D`, 테두리 `#FF8400`, `lucide:trophy` 아이콘 (색상 `#FF8400`)
- 클릭 시 랭킹 페이지(`/league`)로 이동

## 4. 퀴즈 진행

### 4.1 퀴즈팩 구성

- 퀴즈팩 1개당 풀 수 있는 퀴즈 최대 개수: **10개** (어드민에서 제어 가능)
- 퀴즈 유형: `4지선다(multiple)`, `O/X(truefalse)`, `빈칸채우기(choiceblank)` 3가지

### 4.2 퀴즈 진행 규칙

- 보기 순서는 세션마다 달라짐 (`user_quizpack_id` + `session_number` 기반 시드 셔플)
- 진행 중 퀴즈팩 재시작 시 마지막으로 정오답 미기록 퀴즈부터 이어서 진행
- 이미 정오답이 기록된 퀴즈는 앞으로 돌아가 조회만 가능 (재풀이 불가)
- 모든 퀴즈는 정답 확인 전에 힌트 조회 가능, 정오답 결과 후 해설 자동 표시
- 퀴즈 하나씩 풀 때마다 정오답 결과 즉시 표시 및 저장

### 4.3 퀴즈 진행 화면 UI

#### 상단 헤더 (`QuizLayout.tsx`)
- 좌측: 나가기 버튼 (`←`)
- 우측: 획득 중인 Pending XP 배지 (300ms 지연 후 카운트업 애니메이션)

#### 진행 번호 내비게이션 (`QuizNavigation.tsx`)
- 원형 번호 칩 나열 방식

| 상태 | 배경 | 텍스트 |
|------|------|--------|
| 기본 (미풀이) | 투명, 회색 테두리 | 회색 |
| 현재 퀴즈 — 정오답 전 | `#2D2D2D` | `#FF8400` |
| 현재 퀴즈 — 정답 | `#2D2D2D` | `#38D2E3` |
| 현재 퀴즈 — 오답 | `#2D2D2D` | `#FB84C5` |
| 이전 퀴즈 — 정답 | `#38D2E3` | `#2D2D2D` |
| 이전 퀴즈 — 오답 | `#FB84C5` | `#2D2D2D` |

#### 4지선다 보기 색상 (`MultipleChoice.tsx`)

| 상태 | 배경 | 테두리 | 번호 배지 배경 | 텍스트 |
|------|------|--------|--------------|--------|
| 기본 | 흰색 | `#E5E7EB` | `#E5E7EB` | `#374151` |
| 선택 | 흰색 | `#FF8400` | `#2D2D2D` | `#374151` |
| 선택한 정답 | `#DAF5FF` | `#38D2E3` | `#38D2E3` | `#0E7490` |
| 선택 안한 정답 | 흰색 | `#38D2E3` | `#38D2E3` | `#0E7490` |
| 선택한 오답 | `#FEE6F3` | `#FB84C5` | `#FB84C5` | `#BE185D` |

#### O/X 퀴즈 (`TrueFalse.tsx`)
- 거대한 O, X 아이콘만 정사각형 박스 형태로 중앙 배치 (보기 텍스트 없음)
- 기본: 흰색 배경, 회색 테두리/아이콘
- 선택: 연한 오렌지 배경, 오렌지 테두리/아이콘
- 정답: 연한 시안 배경, 시안 테두리/아이콘
- 오답: 연한 핑크 배경, 핑크 테두리/아이콘

#### 빈칸채우기 (`ChoiceBlank.tsx`)
- 지문 안 빈칸: 둥근 인라인 입력창 형태 (`borderRadius: 9999px`)
- 하단 보기 버튼: 캡슐 형태, 좌측 정렬
- 선택 완료: 오렌지 테두리/텍스트
- 정답: 시안 테두리/텍스트
- 오답: 핑크 테두리/텍스트

#### 힌트/정오답 영역 (`QuizContent.tsx`)
- 좌측: `💡 힌트` 텍스트
- 우측: 시안 `O 정답입니다.` 또는 핑크 `X 오답입니다.`
- 해설: 정오답 후 힌트 아래 자동 표시

#### 하단 액션 버튼 (`QuizActions.tsx`)
- 답 입력 전: 연회색 배경, 비활성화 느낌, "정답 확인"
- 답 입력 후: `#2D2D2D` 배경, `#FF8400` 텍스트, "정답 확인"
- 정오답 확인 후: `#FF8400` 배경, 흰색 텍스트, "다음 퀴즈"

### 4.4 퀴즈 완료 시 옵션

- **다시 풀기**: 해당 퀴즈팩을 1번 퀴즈부터 새로 풀기 (이전 답변 복원 없음)
- **결과 보기**: 마지막으로 완료한 시점의 퀴즈별 정오답 결과 조회 (1번부터 순서대로, 별점 입력 불가, 데이터 저장 없음)

## 5. 퀴즈팩 완료 화면

### 5.1 표시 내용

- 퀴즈팩 번호 및 완료 메시지
- 정답률 원형 차트 (애니메이션 카운트업, 도넛 형태)
- 결과 요약 카드: 총 퀴즈 수 / 정답 수 / 획득 XP
- 걸린 시간 (좌측 하단)
- 결과 보기 버튼 (우측 하단, `SearchCheck` 아이콘)
- 별점 입력 영역 (1~5점, 입력 안 해도 CTA 클릭 가능)

### 5.2 액션 버튼

- **다음 퀴즈팩 시작**: `#2D2D2D` 배경, `#FF8400` 텍스트
- **홈으로 가기**: 연회색 배경, 회색 텍스트
- 모든 버튼 호버 시 위로 뜨는 효과 + 그림자 효과

### 5.3 별점 처리

- 별점은 히스토리 방식으로 저장 (퀴즈팩 N번 완료 시 N개의 레코드 생성)
- 퀴즈팩 카드에 표시되는 별점 = 전체 사용자 별점 평균
- 결과 보기 모드에서는 별점 입력 불가

### 5.4 콘페티 효과

- 완료 화면 렌더링 직후 콘페티 효과 1회 발동 (`canvas-confetti` 라이브러리)
- 커스텀 훅 `useConfetti`로 모듈화

### 5.5 후원 팝업 (`SponsorDialog`)

- packId % 4 === 0 인 경우: "다음 퀴즈팩 시작" 클릭 시 후원 팝업 노출 후 이동
- packId % 3 === 0 인 경우: "홈으로 가기" 클릭 시 후원 팝업 노출 후 이동

## 6. XP 시스템

### 6.1 XP 획득 구조

퀴즈 1개 정오답 체크 시 XP 계산:

- **기본 XP**: 난이도별 기준값 (difficulty_id 기반)
- **콤보 보너스**: 연속 정답 카운트(`comboCount`) 기반 추가 XP
- **힌트 차감**: 힌트 사용 시 XP 감소
- **회차별 배율** (`completed_count` 기준):
  - 0회 완료 (1회차): ×1.0 (100%)
  - 1회 완료 (2회차): ×0.5 (50%, 반올림)
  - 2회 이상 완료 (3회차+): ×0.0 (XP 없음)

### 6.2 XP 처리 흐름

```
퀴즈팩 진입 시:
  1. completed_count 확인
  2. completedCount >= 2 → 토스트 팝업 "두 번 이상 완료한 퀴즈팩은 XP를 획득할 수 없습니다." + XP 배지 회색 비활성화

퀴즈 정오답 체크 시:
  1. XP 계산 → pending_xp에 누적 (user_quizpacks.pending_xp)
  2. 마지막 퀴즈 완료 시 XP 확정 (earned_xp로 이전, 누적 정답률 갱신)

퀴즈팩 완료 후:
  1. users.total_xp 업데이트
  2. users.weekly_xp 업데이트
```

### 6.3 XP UI 위치

| 화면 | 위치 | 데이터 | 애니메이션 |
|------|------|--------|-----------|
| 홈 / 마이페이지 | 헤더 아래 중앙 고정 플로팅 모달 | `users.total_xp` | 카운트업 (홈 진입 시 변화량만큼) |
| 퀴즈 진행 화면 | 헤더 우측 | 현재 세션 `pending_xp` | 300ms 지연 후 카운트업/다운 |
| 퀴즈팩 카드 (진행 중) | 카드 우측 하단 | `pending_xp` | 없음 |
| 퀴즈팩 카드 (완료) | 카드 우측 하단 | `earned_xp` | 없음 |
| 완료 화면 | 결과 요약 카드 우측 | `earned_xp` | 정답률 차트와 동일 타이밍 카운트업 |

### 6.4 타이틀(칭호) 시스템

- `users.total_xp` 기준으로 타이틀 자동 부여
- 타이틀 목록: `total_xp_titles` 테이블 관리
- 마이페이지, 랭킹 카드에 타이틀 뱃지 아이콘 + 타이틀 명칭 표시

## 7. 주간 리그 & 랭킹

### 7.1 리그 기간

- 매주 월요일 00:00 KST 시작 ~ 다음 주 월요일 00:00 KST 종료
- 매주 월요일 00:00에 모든 사용자 랭킹 초기화 (`weekly_xp` 리셋)

### 7.2 랭킹 기준 (동점자 처리 순서)

| 순위 | 기준 |
|------|------|
| 1차 | 주간 XP (`users.weekly_xp`) 높은 순 |
| 2차 | 완료한 퀴즈팩 개수(unique) 많은 순 |
| 3차 | 퀴즈팩 완료 횟수 합계 많은 순 |
| 4차 | 퀴즈팩 평균 정답률 (`users.quizpack_avrg_correct`) 높은 순 |
| 최종 | 닉네임 오름차순 (알파벳 → 가나다) |

### 7.3 랭킹 참여 조건

- 주간 XP가 1 이상인 사용자만 랭킹 대상 (`weekly_xp > 0`)
- 퀴즈팩 완료 후 XP 1 이상 획득 시 자동 참여

### 7.4 랭킹 페이지 (`/league`)

#### 헤더 영역 (상단 고정)
- 뒤로가기(`←`) + 도움말(`CircleHelp`) + 새로고침(`RefreshCw`) 버튼
- 타이틀: "주간 리그 랭킹" + 서브타이틀: "Weekly Leaderboard" (`#A1A1A1`, 11px)
- 리그 기간 캡슐 배지: 배경 `#2D2D2D`, 텍스트 `#FF8400`, 형식 "YY.MM.DD ~ YY.MM.DD"

#### 상단 고정 영역 (스크롤 시에도 유지)
- 1위 ~ 3위 랭커 카드

#### 스크롤 영역
- 4위 이하 랭커 카드 + 내 순위 동적 포커싱

#### 랭커 목록 노출 규칙
- 기본 20명 노출 (전체 참가자 < 20명이면 전체 노출)
- 내 주간 XP > 0 이고 20위 이내: 1위 ~ 20위 연속 노출
- 내 주간 XP > 0 이고 21위 밖: 1~3위 고정 + 내 주변 상하위 17명
- 내 주간 XP = 0: 상위 1~20위만 노출
- 최하위권 예외 처리: 내 밑이 8명 미만이면 부족한 수만큼 위쪽 랭커 추가

#### 랭커 카드 표시 정보 (`RankItem`)
- **좌측**: 순위 (1~3위: 메달 SVG 배경 + 흰색 숫자, 4위 이하: 숫자만)
- **중앙**: 닉네임 (Bold) + 타이틀 캡슐 배지 (`#2D2D2D` 배경, 흰색 텍스트)
- **우측**: 주간 XP (상단, 오렌지, Bold, 큰 폰트) + 누적 XP (하단, 회색, 작은 폰트)
- **조회 사용자 강조**: 카드 테두리 `#FF8400`

#### 랭커 카드 툴팁
- **데스크톱**: 마우스 호버 시 노출 / 마우스 이탈 시 제거
- **모바일**: 터치 시 토글
- **툴팁 내용** (좌측부터):
  - 완료한 퀴즈팩 개수 (`Box` 아이콘 + 다크 배지)
  - 총 완료 횟수 (`CircleCheckBig` 아이콘 + 다크 배지)
  - 퀴즈팩 평균 정답률 (텍스트, 소수점 1자리)
- **툴팁 디자인**: 흰색 배경, `rounded-2xl`, `shadow-lg`, 하단 중앙 말풍선 꼬리

#### 빈 랭킹 상태
- 이번 주 참가자 0명 시: "아직 리그에 참여한 사용자가 없습니다. 가장 먼저 퀴즈팩을 완료하고 1위에 등극하세요!"

### 7.5 리그 관련 팝업

## 9. 비즈니스 규칙 & 제약 조건

### 퀴즈팩 순서 변경 시 주의사항

운영 중 퀴즈팩 순서 변경은 사용자 경험에 영향을 줄 수 있으므로 아래 정책을 따른다.

- ✅ **권장**: 새 퀴즈팩을 목록 마지막에 추가 (`closed` 상태로 배포)
- ❌ **지양**: 이미 진행 중이거나 완료된 퀴즈팩의 순서를 변경하여 기존 해금 체계를 혼란에 빠뜨리는 것

순서 변경 시 영향:
- `closed` 퀴즈팩: 위치에 따라 즉시 `opened`로 변환될 수 있음
- `opened` 퀴즈팩: 뒤로 이동 시 `closed`로 잠길 위험
- `in_progress` 퀴즈팩: DB 기록 우선으로 진행 상태 유지, 단 UI 순서 역전 현상 발생 가능
- `completed` 퀴즈팩: 이동 위치의 다음 퀴즈팩을 자동 `opened`로 변환
