# 어드민 사이트 개발 방식 제언

## 1. 개요
"모두의 헌법" 서비스의 효율적인 관리를 위한 어드민 사이트 구축 방향을 검토합니다.

## 2. 추천 방식: 현재 프로젝트 내 통합 개발 (`/admin` 경로 활용)

아론의 관점에서 **현재 프로젝트 내부에서 `/admin` 경로를 사용해 개발하는 것**을 강력히 추천합니다.

### 추천 이유
1. **코드 및 데이터 타입 재사용**: Supabase의 테이블 스키마와 TypeScript 타입을 별도 설정 없이 즉시 공유할 수 있어 개발 속도가 2배 이상 빠릅니다.
2. **관리 일원화**: 환경 변수(API Keys), Supabase 설정, 공통 유틸리티 등을 한 곳에서 관리할 수 있어 유지보수가 매우 편리합니다.
3. **인증 연동**: 이미 구현된 Supabase Auth 시스템을 활용하여 '관리자 권한' 체크 로직만 추가하면 즉시 적용 가능합니다.
4. **배포 효율성**: 별도의 도메인이나 서버 설정 없이 기존 Vercel/Netlify 등 배포 환경을 그대로 활용할 수 있습니다.

## 3. 구현 전략
Next.js의 App Router 기능을 활용해 다음과 같이 구조를 잡습니다.
- `src/app/admin/`: 어드민 전용 레이아웃과 페이지 구성
- `src/components/admin/`: 어드민 전용 UI 컴포넌트
- `src/middleware.ts`: 어드민 경로에 대한 관리자 권한 체크 로직 적용

## 4. 제이슨에게 드리는 의견
제이슨, 현재 프로젝트는 Supabase와 Next.js로 아주 잘 세팅되어 있습니다. 프로젝트를 분리하면 초기 세팅과 데이터 타입 동기화에 불필요한 시간이 소요됩니다. **"가장 빠르고 안정적인 방법"**은 현재 프로젝트 내에 어드민 공간을 만드는 것입니다.
