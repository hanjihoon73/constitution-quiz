# 커스텀 로컬 웹폰트 적용 기획서 (PRD)

## 1. 목적
기존 CDN 방식의 웹폰트를 제거하고, 제이슨 님이 직접 준비해주신 무료 커스텀 폰트(TTF 파일)를 프로젝트의 공식 폰트로 내장하여 최적화 및 안정성, 오프라인 및 빌드 가용성을 높이고자 합니다.

## 2. 요구사항
- **Next.js 폰트 최적화**: Next.js의 내장 기능인 `next/font/local`을 활용해 빌드 시점에 최적화.
- **빠른 로딩**: 글램 폰트 및 로컬 변수 지원 구문을 사용하며, 깜빡임 방지용 `display: "swap"` 최적화를 적용.
- **적용 대상**: 전체 프로젝트 텍스트.

## 3. 기대 효과
- 외부 CDN 스크립트 장애 위험 원천 차단.
- 앱 로딩 속도 강화 및 빌드 프로세스 내 통일성.
