## 3. 테스트 시나리오

실제 앱의 라우트 구조와 예상 사용자 동선을 기반으로 4가지 시나리오를 구성했습니다.

| 시나리오 | 가중치 | 흐름 |
|----------|--------|------|
| 비로그인 사용자 흐름 | 40% | `/` → `/login` |
| 퀴즈 페이지 접속 | 30% | `/` → `/quiz/1` → `/quiz/2` |
| 리그 랭킹 페이지 조회 | 20% | `/` → `/league` |
| 프로필 페이지 조회 | 10% | `/profile` |

> **참고**: 퀴즈, 리그, 프로필 페이지는 인증이 필요한 페이지이므로, 비인증 상태에서는 307 리다이렉트가 반환됩니다. 두 상태코드(200, 307)를 모두 정상으로 허용했습니다.

## 4. 테스트 실행 방법

### 사전 준비

```bash
# Artillery 글로벌 설치 (최초 1회)
npm install -g artillery
```

### 실행

```bash
# 기본 실행 (터미널 출력만)
artillery run tests/quiz-load-test.yml

# JSON 리포트 저장 포함 실행 (권장)
artillery run --output report.json tests/quiz-load-test.yml
```

> **참고**: `artillery report report.json` 명령(HTML 리포트 생성)은 최신 Artillery 버전에서 폐기되어 지원되지 않습니다. JSON 결과를 시각화하려면 [Artillery Cloud](https://app.artillery.io)를 사용하거나 터미널 출력 결과를 직접 확인하세요.
